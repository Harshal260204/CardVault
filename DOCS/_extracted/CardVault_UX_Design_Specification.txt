Progressive Disclosure
Show exactly what the user needs for their current task — not everything CardVault can do.
Advanced settings, bulk operations, and export functions live in context menus or overflow panels.
Qualification tags (HOT/WARM/COLD) must be one-tap accessible on the exhibitor capture screen — zero sub-menus.
Offline-First Trust
The UI must never appear "broken" when offline. Show explicit offline states, queued counts, and expected sync times.
Offline badges use amber (#F59E0B) — not red, which implies error. Offline is a valid operating state.
All captures completed offline display a subtle sync-pending indicator that disappears post-sync without disrupting workflow.

1.4 Enterprise Usability Philosophy
Every user role has a tailored experience — Super Admin, Org Admin, Event Manager, Sales Rep, Exhibitor.
UI surfaces only the controls relevant to the authenticated role. Role-irrelevant actions are hidden, not disabled.
Bulk operations are always visible when a table is present. Admins expect to act on multiple records simultaneously.
Confirmation dialogs are reserved for destructive or irreversible operations only. Non-destructive actions confirm via toast.
Loading states are always informative: skeleton loaders mirror the actual content layout.

Returning from background always shows last capture state, never resets to home screen.
1.6 Accessibility Philosophy
WCAG 2.1 AA compliance is baseline. WCAG 2.1 AAA for critical flows (OCR review, session management, admin controls).
Color is never the sole differentiator — always paired with shape, icon, or label.
All interactive components support keyboard navigation with visible focus rings (2px offset, brand blue color).
Screen reader support via semantic HTML (web) and accessibilityLabel + accessibilityRole (React Native).
Minimum 4.5:1 contrast ratio for body text; 3:1 for large text and UI components.
1.7 How Speed and Confidence Shape UI
CardVault's primary value moment is the 10–15 second window on an event floor when a rep scans a business card. Every design decision for mobile capture must optimize for this window.
Confidence = Predictability. Buttons don't move between states. Primary actions are always at the bottom. OCR results appear in a consistent card layout every time.
Speed = Zero Friction. No login prompts mid-event (persistent auth). No multi-step forms for capture. OCR pre-fills everything — user only corrects.
Enterprise Trust = Visual Calm. No anxiety-inducing reds unless something actually failed. Muted palette with purposeful accent use. Dense but breathable layouts.

2.6 Accessibility & Contrast Notes
All color combinations in this palette meet WCAG 2.1 AA. Critical flow components (OCR review, session join, qualifier selection) target AAA.
brand-600 (#1A56DB) on white background: 5.74:1 — passes AA for body text ✓
neutral-950 (#0F172A) on white: 19.1:1 — passes AAA ✓
neutral-500 (#64748B) on white: 4.6:1 — passes AA for body text ✓
warning-500 on white: 3.2:1 — use only for large text or icons (not body copy)
Never use warning/error colors as background for text smaller than 18px bold / 24px regular.
2.7 Mobile Visibility Considerations
On OLED screens, brand-800 (#1E3A5F) renders with exceptional depth — use as sidebar background on mobile dark mode.
Avoid pure black (#000000) for text — neutral-950 (#0F172A) provides OLED-safe softness.
Qualifier chip colors (HOT/WARM/COLD) use 2px colored left border in addition to background for color-blind accessibility.
Outdoor event use: All primary capture UI elements maintain minimum 7:1 contrast for bright sunlight legibility.

3.4 Typography Rules
Line length: 65–80 characters max for body copy. Use max-width constraints on content columns.
Never set body text below 13px on web, 12sp on mobile.
Table cells: body-md (14px) with 1.4 line height. Dense but never cramped.
Form labels: body-sm (13px), weight 500 (medium), neutral-700. Always above the field, never inside as placeholder-only.
Numeric data in tables always uses JetBrains Mono for alignment and digit clarity.
Headings use Inter with optical size adjustments — do not fake weight with letter-spacing.
Letter-spacing: body copy 0. Headings at display sizes: -0.02em tracking for tighter premium feel.
Uppercase text: Use sparingly — qualifier labels, table column headers, badge labels only. Never for body copy.

8.3 Rules for Motion
Motion must earn its place. Every animation either communicates state change, directs attention, or provides confirmation. If it does none of these, remove it.
Respect prefers-reduced-motion: all animations have a no-motion fallback (instant state change)
Never animate layout shifts that move content a user is reading
Loading skeletons should shimmer no faster than 1.5s cycle — faster feels anxious
Success animations play once — never loop
Error states never animate repeatedly — static treatment after initial attention-getting

15. UI Engineering Guidelines
Practical guidance for implementation teams. Token-driven theming, shared design system between web and mobile.
15.1 TailwindCSS Architecture
Concern
Approach
Custom tokens
Extend tailwind.config.js with design token values from Section 11. Never hardcode hex values in JSX.
Component variants
Use cva (class-variance-authority) for component variant management. Define base + variant + compoundVariant.
Dark mode
class strategy (not media) for user-toggled dark mode. Apply dark: prefixes from design token dark values.
Font loading
next/font for Inter + JetBrains Mono with display: swap. Subset to Latin only.
Animation
tailwindcss-animate plugin + custom keyframes in tailwind.config for branded animations.
Responsive utilities
Mobile-first: base = mobile, md/lg/xl prefixes for larger. Never desktop-first.
15.2 React Native Styling Approach
Concern
Approach
Styling library
NativeWind (Tailwind for React Native) for web-mobile token consistency
Design tokens
Shared tokens package (@cardvault/tokens) consumed by both web + mobile
Platform differences
Platform.select() for shadow (iOS boxShadow vs Android elevation)
Typography
Expo Google Fonts for Inter. Dynamic font scaling: useWindowDimensions + scale factor
Safe areas
react-native-safe-area-context throughout. Never hardcode status bar heights.
Gesture handling
react-native-gesture-handler for swipe qualifiers, drag-to-sort.
Animation
react-native-reanimated 3.x for all animations. Never Animated API for complex flows.
